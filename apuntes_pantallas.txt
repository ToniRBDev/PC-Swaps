# Estructura proyecto React


## Pantallas y funcionalidades:

### Pantalla 00:
	- Landing page que presenta las funciones de la aplicacion, donde muestra una lista de cards de artículos anunciados
	- También motivará a los que la visiten a crearse una cuenta. Ejemplos:
		- Destacar la especialización de la aplicación en ese sector
		- Reutilización de componentes informáticos
		- Ahorro en componentes
	- Tendrá el mismo estilo de colores que el resto de la aplicación
	- Al fondo de la landing page, habrá:
		- ¿Quieres unirte a nuestra comunidad? y un botón de crear cuenta
		- ¿Ya eres miembro? y un botón de iniciar sesión

### pantalla 01: 

	Login o creacion de cuenta (Utilizar la misma pantalla, ya que son similares), sólo modificar los input 
	Campos pantalla creación de cuenta:
		- nombre: Texto (String). No puede estar vacío.
		- apellidos: Texto (String). No puede estar vacío.
		- dni: Texto (String). No puede estar vacío.
		- correoElectronico: Texto (String). Debe tener formato de email válido.
		- fechaNacimiento: Fecha (LocalDate). Debe ser una fecha pasada.
		- nombreUsuario: Texto (String). No puede estar vacío.
		- password: Texto (String). Mínimo 8 caracteres.
	Campos pantalla inicio de sesión
		- Correo electrónico
		- Contraseña
	La imagen se le asignará una por defecto con las iniciales que luego podrá modificar en la pestaña modificar perfil.
	
	
	
### pantalla 02: pantalla home
	- Componente navBar siempre visible en todas las pantallas, excepto la landing page
	Funcionalidades de los botones de la navBar:
		- Ver mis anuncios: Nos va a enrutar a la page 06, Mis anuncios
		- Publicar ununcio: Nos va a enrutar a la page 03, Creacion de anuncio
		- Mis mensajes: Nos va a enrutar a la page 07, Mis conversaciones (El botón mis mensajes mostrará una notificá con un punto rojo que si hay mensajes pendientes)
		- Barra de búsqueda: Se mostrarán las coincidencias con lo introducido por el usuario en la sección últimos artículos, que será 
			renombrada a RESULTADOS:
		- Botón Modificar mis datos: Nos va a enrutar a la page 04 Modificar mis datos
		- Botón modificar contraseña: Nos va a enrutar a la page 09 Modificar contraseña
		- Botón cerrar sesión: Cerrará la sesión del usuario y nos llevará a la pantalla 01: Iniciar sesión
		- Botón eliminar cuenta: Nos saldrá una ventana emergente: ¿Estás seguro de eliminar tu cuenta), tendrá 2 botones: Sí y No
	- Iconos de la HomePage: buscarán todos los artículos de la categoría que indica el icono seleccionado por el usuario
	- Botón de ver todo: 
		- Se ampliará la cantidad de artículos mostrados en la sección últimos artículo subidos mostrando 4 filas por página, se mostrará la paginación
		- Se dejarán de mostrar los artículos en seguimiento
		- 
	- Sección últimos artículos subidos:
		- Se mostrará una fila con los últimos artículos subidos
		- En cuanto se relice una búsqueda en la barra de búsqueda o filtro por categoría haciendo click en el icono, automáticamente se renombrará el nombre de la categoría y se mostrarán los productos filtrados 
		
	- Sección Seguimiento:
		- Se mostrarán todos los artículos en seguimiento, en caso de que haya más de 3 filas, se mostrará paginación
		- Esta sección se ocultará automáticamente cuando se realice un filtro por categoría o filtrado por la barra de búsqueda
		



## Pages

	### Page 01: Page raíz
	- Será la page home, se ha descrito en la sección anterior, no incluye la navBar, que estará siempre visible independientemente en qué page estemos situados
	- Todas las pages menos esta, l deben de tener botón del volver atras, como la page ya creada 02 Visualizacion de anuncio
		
		
	### Page 02: Visualizacion anuncio
	- Básate en la página de muestra: /PC-Swaps\docs\pantallas_4\12_visualizacionAnuncio.html, (recuerda que la barra de navegacion no)
	- Va a recibir los siguientes datos del backend para mostrar:
		- categoría
		- marca
		- modelo
		- estado
		- precio
		- descripcion
		- imagen
		- fecha de publicacion
		- numero de visitas
	- Botones:
		- Iniciar char con el vendedor: Enrutará a la page chat
		- Mostrar información del vendedor: Enrutará a la page datos del vendedor
		- Añadir a seguimiento o Eliminar de seguimiento si ya lo tenemos en seguimiento
			 
		
		
	

	### Page 03: Creacion de anuncio
	- Básate en la página de muestra: "PC-Swaps/docs/pantallas_4/05_creacionDeAnuncio.html"
	- Vas a introducir los siguientes datos:
		- marca
		- modelo
		- desplegable categoria
		- Desplegable seleccionar estado artículo
		- Precio
		- Descripcion del vendedor
		- Imagen 
	- Botón de creación de anuncio, que lo creará y enviará los datos al backend
		- Si todo sale bien, saldrá una notificación Anuncio creado!!
		- Si algo sale mas, saldrá una notificación No se ha podido crear el anuncio!!
	- Una vez del éxito de la creación del anuncio, volvemos automáticamente a la página home
	- También debe de tener un botón volver a la página de inicio
	
	

	### Page 04:
	- Modificar los datos de mi perfil
	- Estos son los datos que se modificaran y se enviarán al backend
	
	- @NotBlank(message = "El nombre de usuario es obligatorio")
        String nombreUsuario,

        @NotNull(message = "La fecha de nacimiento es obligatoria")
        @Past(message = "La fecha de nacimiento debe ser una fecha pasada")
        LocalDate fechaNacimiento,

        String direccion,

        String numTelefono,

        String imagenUsuario

	

	### page 05: Mostrar datos del vendedor 
	"PC-Swaps\docs\pantallas_4\08_datosContactoVendedor.html"
	Mostrará los siguientes datos que reciba del backend:
		Long idUsuario, (El id usuario no se va a mostrar, es solo para busqueda)
        String nombreUsuario,
        String correoElectronico,
        String direccion,
        String numTelefono,
        String imagenUsuario

		El botón atrás llevará de nuevo al anuncio
		También tendrá el botón Iniciar chat, que redireccionará al mismo lugar que el botón de la visualización del anuncio.
