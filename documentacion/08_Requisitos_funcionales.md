## 08. Catálogo de requisitos

### Requisitos funcionales

#### 1.	Búsqueda de articulos

    1.1. Búsqueda de articulos mediante una barra de búsqueda.

    1.2. Búsqueda de articulos haciendo click en el icono del articulo. Automáticamente se realizará una búsqueda de todos los articulos de esa categoría.


#### 2.	Registrarse como usuario

    2.1. Formulario de datos.
        2.1.1. Nombre
        2.1.2. Apellidos
        2.1.2. DNI
        2.1.3. Correo electrónico.
        2.1.4. Fecha de nacimiento.
        2.1.5. Dirección.
        2.1.6. Número de teléfono.
        2.1.7. Nombre de usuario.
        2.1.8. Contraseña.


#### 3.	Identificarse como usuario registrado

    3.1. Iniciar sesión.

    3.2. Cerrar sesión.

    3.3. Recuperación de contraseña.


#### 4.	Perfil usuario

    4.1. Modificar datos personales.

        4.1.1. Nombre
        4.1.2. Apellidos
        4.1.3. DNI
        4.1.4. Fecha de nacimiento
        4.1.5. Dirección
        4.1.6. Número de teléfono

    4.3. Modificar datos inicio sesión 

        4.3.1. Nombre de usuario.
        4.3.2. Contraseña.


#### 5.	Gestión de venta de articulos.

    5.1. Publicar anuncio articulo

    5.1.1. Datos del artículo
        5.1.1.1. Marca
        5.1.1.2. Modelo
        5.1.1.3. Estado 
        5.1.1.4. Precio 
        5.1.1.5. Descripción 
        5.1.1.6. Imagen
        5.1.1.7. Fecha de publicación 

    5.2. Renovación de anuncios. Los anuncios deber ser renovados frecuentemente.

    5.3. Eliminar articulo.


#### 7. Mensajería entre usuarios.

    7.1. Posibilidad de enviar mensajes entre usuarios sobre un articulo determinado.


### Requisitos no funcionales


#### 1. Seguridad y autenticación

    1.1. Integridad de los datos de los usuarios.


#### 2.	Usabilidad

    2.1. Diseño responsivo, que se adapta a todo tipo de dispositivos. (TODO: PENSAR SI SE DA TIEMPO O NO)

    2.2. Diseño intuitivo y fácil de usar.


#### 3.	Fiabilidad

    3.1. Soporte para la interacción de un gran número de usuarios al mismo tiempo.


#### 4.	Accesibilidad

    4.1. Cumplimiento de los estándares mínimos para el acceso a todo tipo de personas.


#### 5. Escalabilidad

    5.1. La aplicación debe permitir futuras ampliaciones.

    5.2. Soporte para gran cantidad de usuarios registrados.
