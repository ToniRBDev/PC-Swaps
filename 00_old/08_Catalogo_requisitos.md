## 08. Catálogo de requisitos

### Requisitos funcionales

#### 1.	Búsqueda de productos


    1.1. Búsqueda de productos mediante una barra de búsqueda. Se establecerá un filtro de productos acorde a lo que haya introducido el usuario.

    1.2. Búsqueda de productos haciendo click en la imagen del producto. Automáticamente se realizará una búsqueda de todos los productos de ese tipo. 

        1.2.1.	Imagen de Placa base.
        1.2.2.	Imagen de Procesador.
        1.2.3.	Imagen de tarjeta gráfica. 
        1.2.4.	Imagen de memoria RAM.
        1.2.5.	Imagen de fuente de alimentación.
        1.2.6.	Imagen de disco duro HDD.
        1.2.7.	Imagen de disco duro SSD.
        1.2.8.	Imagen de disipador de calor.
        1.2.9.	Imagen de caja de PC.
        1.2.10.	Imagen de PC usado.
        1.2.11.	Imagen de portatil.
        1.2.12.	Imagen de periféricos.

    1.3. Filtrar producto por determinadas características


#### 2.	Registrarse como usuario

    2.1. Formulario de datos.


#### 3.	Identificarse como usuario registrado

    3.1. Iniciar sesión.

    3.2. Cerrar sesión.

    3.3. Recuperación de contraseña.


#### 4.	Perfil usuario

    4.1. Historial de pedidos.

    4.2. Modificar datos personales.

        4.2.1. Nombre.
        4.2.2. Apellidos.
        4.2.3. Fecha de nacimiento.
        4.2.4. Número de identificación.
        4.2.5. Dirección.
        4.2.6. Número de teléfono.

    4.3. Modificar datos inicio sesión

        4.3.1. Nombre de usuario.
        4.3.2. Contraseña.

    4.4. Métodos de pago

        4.4.1. Tarjeta bancaria.
            4.4.1.1. Añadir datos de nueva tarjeta.
            4.4.1.2.	Modificar datos tarjeta actual.
        4.4.2.	Paypal.


#### 5.	Gestión de venta de productos.

    5.1. Publicar anuncio producto

        5.1.1. Formulario detallado de datos del producto.
        5.1.2. Imágenes de producto.

    5.2. Renovación de anuncios. Los anuncios deber ser renovados frecuentemente.

    5.3. Eliminar producto.

    5.4. Reserva de productos.

    5.5. Confirmación de venta de producto.


#### 6.	Comprar producto.

    6.1. Añadir producto al carrito.

    6.2. Confirmar compra de producto.

    6.3. Confirmación recepción articulo

    6.4. Confirmación funcionamiento del artículo.


#### 7. Mensajería entre usuarios.

    7.1. Chat para contactar con el vendedor de un producto.


#### 8. Pago

    8.1. Seleccionar forma de pago

    8.2. Añadir nueva forma de pago

    8.3. Opción de guardar forma de pago en mis métodos de pago.

#### 9. Valoraciones

    9.1. Valoración de comprador a vendedor

    9.2. Valoración de vendedor a comprador

    9.3. Valoración general de la aplicación


#### 10.	Notificaciones

    10.1. Notificaciones por correo electrónico 

        10.1.1. Venta de producto.
        10.1.2. Mensaje de usuario.
        10.1.3.	Renovar anuncio antes de X h.
        10.1.4.	Notificaciones de la aplicación
        10.1.5.	Venta de producto.
        10.1.6.	Mensaje de usuario.
        10.1.7.	Renovar anuncio antes de X h.


### Requisitos no funcionales


#### 1. Seguridad y autenticación

    1.1. Integridad de los datos de los usuarios.

    1.2.Seguridad en los pagos.


#### 2.	Usabilidad

    2.1. Diseño responsivo, que se adapta a todo tipo de dispositivos.

    2.2. Diseño intuitivo y fácil de usar.


#### 3.	Fiabilidad

    3.1. Soporte para la interacción de un gran número de usuarios al mismo tiempo.


#### 4.	Accesibilidad

    4.1. Cumplimiento de los estándares mínimos para el acceso a todo tipo de personas.


#### 5. Escalabilidad

    5.1. La aplicación debe permitir futuras ampliaciones.

    5.2. Soporte para gran cantidad de usuarios registrados.
